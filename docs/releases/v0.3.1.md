# rat v0.3.1 release notes

Release scope: Go CLI and embedded Python runtime since `v0.3.0`.

This is a focused Python-runtime hardening release. It fixes a class of failures where user code, native libraries, or subprocesses could write to stdout/stderr and corrupt Rat's internal Python kernel protocol.

## Highlights

- Python kernel control traffic now uses a private protocol channel instead of the process stdout stream.
- Python user stdout/stderr are treated as normal cell output, including output from subprocesses and low-level `os.write(1/2, ...)` calls.
- Subprocess stdin no longer points at Rat's protocol input, so child processes cannot consume kernel control messages.
- Protocol bootstrap details are removed from the Python environment before user code runs, so subprocesses do not inherit them.

## Fixed

### External subprocess output no longer corrupts Python kernels

Previously, Rat's embedded Python kernel used stdout for JSON control messages. Python-level `print()` was captured safely, but child processes inherited the original stdout file descriptor. If a subprocess wrote anything there, Rat could try to parse that text as JSON and fail with errors such as:

```txt
decode python kernel response: unexpected end of JSON input
```

This affected code like:

```python
import subprocess
subprocess.run(["ffplay", "-nodisp", "-autoexit", "preview.wav"])
```

and could also be triggered by direct fd writes:

```python
import os
os.write(1, b"raw stdout\n")
os.write(2, b"raw stderr\n")
```

Rat now separates the Python control protocol from user stdout/stderr. Subprocess output is captured as ordinary run output and the kernel remains alive.

### Child stdin no longer consumes Rat protocol input

Python subprocesses now inherit a safe stdin instead of Rat's protocol input. A child process reading from stdin can no longer steal pending control messages from the kernel.

## Internal

- The Go Python runtime starts the embedded Python kernel with a private localhost protocol socket and a random bootstrap token.
- The Python kernel also supports `RAT_PROTOCOL_IN_FD` / `RAT_PROTOCOL_OUT_FD` for embedders that want to provide private file descriptors.
- Added regression tests for subprocess stdout, subprocess stderr, raw fd writes, child stdin isolation, protocol env cleanup, and kernel liveness after subprocess output.
